# 档案库构建与维护协议 (Project Digital Specimen Manual)
> **版本**：1.0 (Industrial / Deep Sea Edition)
## 第一章：环境搭建 (Environment Setup)
任何想要在本地运行或编辑此博客的人（包括协作者），必须安装以下环境。
### 1. 核心工具
* **VS Code**: 推荐的代码编辑器。
* **Go 语言环境**: [下载链接](https://go.dev/dl/) (必须安装，否则报错)。
* **Git**: [下载链接](https://git-scm.com/download/win) (必须安装，用于下载主题模块)。
* **Hugo Extended 版本**: [下载链接](https://github.com/gohugoio/hugo/releases)
    * **警告**：必须下载文件名带 `extended` 的版本（如 `hugo_extended_..._windows-amd64.zip`）。
    * **安装位置**：将解压出的 `hugo.exe` 直接放入项目根目录（与 `config` 文件夹同级）。
### 2. 启动命令
在 VS Code 终端 (Terminal) 中输入以下命令启动本地预览：

```powershell
./hugo server
```

* 预览地址: http://localhost:1313/my-game-archive/
* 停止服务: 在终端按 Ctrl + C。

## 第二章：写作规范 (Writing Protocol)
为了保持文件整洁，本站严格采用 "Page Bundles" (页面打包) 模式。
1. 新建文章流程
1. 进入目录：content/post/。
1. 新建文件夹：文件夹名即为文章链接名（如 dawang-guide）。
1. 新建文件：在文件夹内创建 index.md。
1. 放入素材：将文章需要的图片（封面、插图）直接拖入该文件夹。
1. 文章头部信息 (Front Matter)
复制以下模板到 index.md 顶部：
```YAML
title: "文章标题"
description: "显示在卡片上的简短摘要"
date: 2025-12-14
slug: "自定义英文链接名"
image: "cover.jpg"       # 封面图，填写同级目录下的文件名
categories:
    - "Game Review"
tags:
    - "High Difficulty"
    - "Aesthetics"
weight: 0                # 普通文章填 0，置顶填 1
```
内容排版
* 插入图片：直接写文件名，不要加斜杠。
```Markdown
![图片描述](screenshot.jpg)
```
* 二级标题：使用 ##，增加文章骨架感。
* Bilibili 视频：使用专用 Shortcode（自带工业风边框）。
```Markdown
{{< bilibili BV号 >}}

示例：{{< bilibili BV1aSmiBrEDa >}}
```

## 第三章：特殊板块维护
### 1. 首页公告栏 (Notice)
* 位置: content/post/notice.md (或同名文件夹下的 index.md)
* 操作: 修改内容并保存即可。
* 机制: 依靠 weight: 1 属性强制置顶。如果不想显示，删除该属性或将其改为 draft: true。
### 2. 友情链接 (Links)
* 位置: content/page/links/index.md
* 格式: 在 links: 列表下添加新的 block。
### 3. 留言板 (Guestbook)
* 位置: content/page/guestbook/index.md
* 配置: 评论系统基于 Giscus (GitHub Discussions)。如需修改仓库配置，请编辑 config/_default/params.toml 中的 [comments.giscus] 部分。

## 第五章：常见问题 (Troubleshooting)

Q：为什么运行 ./hugo server 报错？

A：请检查是否安装了 Extended 版本的 Hugo，以及电脑是否安装了 Go 和 Git。安装后必须重启 VS Code。

Q：为什么图片显示不出来？

A： 1. 检查图片是否在文章对应的文件夹内（Page Bundle 模式）。2. 检查 Markdown 里引用时是否多了斜杠（正确：image.jpg，错误：/image.jpg）。

Q：为什么 B 站视频没有边框？

A：确保使用了 {{< bilibili ... >}} 语法，而不是直接粘贴 iframe 代码。
